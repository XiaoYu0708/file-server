# File Server

[![Docker Hub](https://img.shields.io/badge/docker-xiaoyu0708/file--server-2496ED?logo=docker)](https://hub.docker.com/r/xiaoyu0708/file-server)

區域網路檔案交換工具。不需安裝任何 App，打開瀏覽器即可在多個裝置間互傳檔案。

## 運作方式

```
                 ┌──────────────────┐
                 │  Windows Server  │
                 │  (背景執行)      │
                 └──────┬───────────┘
                        │
          ┌─────────────┼─────────────┐
          │             │             │
     ┌────▼────┐  ┌────▼────┐  ┌────▼────┐
     │ Phone A │  │ Phone B │  │   PC    │
     │ Safari  │  │ Safari  │  │ Chrome  │
     └─────────┘  └─────────┘  └─────────┘
```

伺服器只負責轉發與暫存，所有操作都在瀏覽器上完成。

## 使用方式

### 1. 啟動伺服器 (Docker)

```bash
docker run -d -p 5000:5000 --name file-server xiaoyu0708/file-server
```

終端機會顯示 QR Code 與伺服器網址。檔案會暫存在容器內的 `/app/data/rooms/`，容器刪除後資料會消失。若要持久化儲存，加上 volume：

```bash
docker run -d -p 5000:5000 -v /path/to/data:/app/data --name file-server xiaoyu0708/file-server
```

### 或用 Docker Compose（推薦，開機自動啟動）

```bash
docker compose up -d
```

會自動在背景啟動，並在每次開機時自動運行。檔案存放在同目錄的 `data/` 資料夾。

### 2. 各裝置開啟網頁

手機掃 QR Code 或在瀏覽器輸入 `http://<IP>:5000`

### 3. 配對

```
┌─ Phone A ─────────────────────┐     ┌─ Phone B ─────────────────────┐
│  📡 你的配對碼                 │     │  📡 你的配對碼                 │
│                                │     │                                │
│       8  R  Q  2               │     │       A  N  H  M               │
│                                │     │                                │
│  ──── 輸入對方的配對碼 ────    │     │  ──── 輸入對方的配對碼 ────    │
│  [ANHM]          [配對]        │     │  [8RQ2]          [配對]        │
└────────────────────────────────┘     └────────────────────────────────┘
```

Phone A 輸入 Phone B 的碼，或 Phone B 輸入 Phone A 的碼，兩邊即完成配對。

### 4. 互傳檔案

配對後兩邊看到同一個檔案列表，任一邊上傳另一邊立即看到：

```
┌─ Phone A ─────────────────────┐     ┌─ Phone B ─────────────────────┐
│  已與 ANHM 配對    [解除配對]  │     │  已與 8RQ2 配對    [解除配對]  │
│                                │     │                                │
│  📤 上傳                       │     │  📤 上傳                       │
│  ⬆ 點此選取檔案               │     │  ⬆ 點此選取檔案               │
│                                │     │                                │
│  📋 檔案列表                   │     │  📋 檔案列表                   │
│  📄 photo.jpg   1.2MB [下載]   │     │  📄 photo.jpg   1.2MB [下載]   │
│  📄 report.pdf  2.4MB [下載]   │     │  📄 report.pdf  2.4MB [下載]   │
└────────────────────────────────┘     └────────────────────────────────┘
```

## 功能

| 功能 | 說明 |
|------|------|
| **裝置配對** | 每台裝置有獨立配對碼，輸入對方的碼即可配對 |
| **上傳檔案** | 點擊上傳區選取檔案，支援拖曳 |
| **上傳資料夾** | 切換至「資料夾」模式，一次上傳整個目錄 |
| **下載檔案** | 點擊「下載」按鈕直接下載 |
| **下載資料夾** | 自動壓縮為 ZIP 封裝下載 |
| **刪除** | 點擊「刪除」按鈕 |
| **解除配對** | 點擊「解除配對」即時中斷連線 |
| **多組配對** | 支援多組裝置各自獨立配對，互不干擾 |
| **Port 衝突處理** | 若 5000 埠被占用，自動搜尋 5000~5100 可用埠 |

## 伺服器

伺服器僅作為背景中繼，本身無需任何操作：

```
┌─────────────────────────────────────────────────────┐
│                 File Server 已啟動                  │
│                                                     │
│  http://192.168.0.2:5000                            │
│                                                     │
│  █▀▀▄▄ ... QR Code                                 │
│                                                     │
│  手機掃描上方 QR Code 開啟網頁                     │
│  按 Ctrl+C 停止伺服器                              │
└─────────────────────────────────────────────────────┘
```

## 從原始碼建置

```bash
docker build -t file-server .
```

## 網路需求

- 所有裝置需連接**同一台路由器**（同一個區域網路）
- Windows 防火牆需允許 TCP 連線
- 若連線逾時，請檢查分享器是否有啟用「AP 隔離」功能
